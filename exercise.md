# Итоговый проект

## Цель проекта
Развернуть web-приложение для работы в облачной инфраструктуре Yandex Cloud.

---

## Описание проекта
Задания итогового проекта охватывают полный цикл создания и настройки инфраструктуры, установку необходимых инструментов, сборку и развертывание приложения, а также хранение образов в реестре контейнеров.

---

## В рамках итогового проекта вы:

- Соберете простое web-приложение на основании предоставленных данных (описание `Dockerfile`, `docker-compose.yml`).
- Настроите инфраструктуру в Yandex Cloud с использованием Terraform.
- Развернете приложение в облачной среде.

---

## Инструкция по выполнению итогового проекта

Используя инструменты Docker, Docker Compose и Terraform, выполните следующие задания:

### Задание 1. Развертывание инфраструктуры в Yandex Cloud

- Созданы два модуля для Terraform:
  - [Модуль для VPC](https://github.com/DioRoman/ter-final/tree/main/src/modules/yandex-vpc)
  - [Модуль для VM](https://github.com/DioRoman/ter-final/tree/main/src/modules/yandex-vm)
  
- На основе этих модулей формируются виртуальные машины, сеть и подсети:  
  [Основной код проекта](https://github.com/DioRoman/ter-final/tree/main/src)

- Основные шаги:
  - Создайте виртуальные машины (VM).
  - Настройте группы безопасности с открытыми портами:  
    - 22 (SSH)  
    - 80 (HTTP)  
    - 443 (HTTPS)
  - Привяжите группу безопасности к VM.

- Дополнительно опишите:
  - Создание MySQL базы данных в Yandex Cloud.
  - Создание Container Registry.
СДЕЛАНО
---

### Задание 2. Установка Docker и Docker Compose через user-data (cloud-init)

- Используя механизм `cloud-init` (`user-data`), установите Docker и Docker Compose на VM.  
- См. Задание 4 модуля «Виртуализация и контейнеризация» для примеров.

СДЕЛАНО
---

### Задание 3. Описание Dockerfile и сохранение контейнера

- Опишите `Dockerfile` для web-приложения (с мультистадийной сборкой).  
- Соберите образ и сохраните его в Container Registry.  
- См. Задание 4 модуля «Виртуализация и контейнеризация» для схожих примеров.

СДЕЛАНО
---

### Задание 4. Интеграция приложения с базой данных в облаке

- Настройте работу web-приложения в контейнере таким образом, чтобы оно использовало базу данных MySQL, развернутую в Yandex Cloud.

---

### Задание 5*. Хранение паролей в LockBox и интеграция с Terraform

- Поместите пароли от базы данных в LockBox Yandex Cloud.  
- Настройте интеграцию с Terraform для извлечения пароля из LockBox и автоматического использования его при развертывании инфраструктуры.

---

## Чек-лист готовности итоговой работы

- [ ] Инфраструктура в Yandex Cloud описана без хардкода, state Terraform хранится удаленно с подключенным statelocking.
- [ ] Docker и Docker Compose установлены через `cloud-init`.
- [ ] Dockerfile содержит мультистадийную сборку и настроено сохранение образа в Container Registry.
- [ ] Приложения доступны по IP-адресу виртуальной машины (опционально: настроен DNS).
- [ ] Создан Markdown-файл с корректным оформлением, содержащий примеры, скриншоты и ссылки.

---

> По всем вопросам выполнения проекта рекомендуем обратиться к репозиториям и материалам курсовых модулей для дополнительной информации.
